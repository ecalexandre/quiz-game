# Explanations of Sections

## 1. Title

> "What is this project called?"

Simple.

## 2. Description

Answer only one question:

> **What does this program do?**

Not:

> It uses Python.

Nobody cares.

Instead:

> This program allows a user to play a number guessing game.


## 3. Features

Answer:

> **What can the user do?**

Example:
```
-   Add tasks    
-   Remove tasks    
-   View tasks    
-   Exit the application  
```
Very simple.

## 4. Requirements
What does someone need?

Usually:
```
Python 3.13
```
Later he'll add packages.

## 5. Installation

How do I prepare the project?

Eventually he'll learn:
```
git clone  
python -m venv  
pip install
```
For now, keep it simple.

## 6. Usage

Probably the most important section.

Explain:
> "How do I run your program?"

Example:
```
python3 task_manager.py
```
Then explain:
```
The menu appears.
Choose A to add a task.
Choose R to remove one.
Choose Q to quit.
```
Perfect.

## 7. Example

Developers LOVE examples.

Instead of reading paragraphs:
```
===== TASK MANAGER =====
A - Add
V - View
R - Remove
Q - Quit
```
Now they immediately understand.

## 8. Project Structure

Another professional habit.
```
task-manager/
├── README.md
├── task_manager.py
└── requirements.txt
```

Now someone opening the repository immediately understands it.

## 9. Author

Simple.
```
Created by Elliot Alexandre.
```
He'll smile ten years from now seeing that.
